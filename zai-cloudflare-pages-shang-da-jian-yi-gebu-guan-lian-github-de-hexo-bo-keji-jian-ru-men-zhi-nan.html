# 在 Cloudflare Pages 上搭建一个「不关联 GitHub」的 Hexo 博客（极简入门指南）
去年在 GitHub 上建了个博客，本想让 Cloudflare Page 直接连接 GitHub 仓库，后来觉得还是重新弄一个，毕竟使用 Hexo 也很有趣。下面简单介绍如何搭建一个最基础的不关联 GitHub 的博客。



第一步：注册 Cloudflare Page 账号
先去 https://pages.cloudflare.com/ 申请个账号，网上相关教程很多，这里不再赘述。

第二步：下载并安装 Node.js
下载地址：https://nodejs.org/zh-cn/download ，下载完成后一路点击“下一步”，中途无需点击其他框。安装完成后，按 Win + R 组合键，输入 cmd 打开命令提示符窗口，依次输入 node -v 和 npm -v 并回车，如果能返回版本号信息，说明安装成功。

第三步：下载并安装 Git
下载地址：https://git-scm.com/downloads ，同样一路点击“下一步”完成安装。安装完成后，按 Win + R 组合键，输入 cmd 打开命令提示符窗口，输入 git --version 并回车，若能返回版本号信息，则安装成功。

第四步：安装 Hexo 并初始化博客项目
在命令提示符窗口输入 npm install -g hexo-cli ，等待约一分钟左右完成下载。
输入 hexo -v ，若返回 Hexo 版本信息，说明安装成功。
输入 mkdir hexo-blog ，新建一个博客安装位置，这通常只需几秒钟，记住该安装位置信息，后续会用到。
输入 cd hexo-blog 进入该文件夹。
输入 hexo init 生成文件。
输入 npm install 。至此，博客程序已在本地安装完成。
第五步：了解博客项目文件结构
进入博客程序的安装文件夹，会看到以下常用文件（夹）：

_config.yml：Hexo 全局配置文件，用于配置博客标题、主题、部署信息等。
source/_posts/ 文件夹：用于存放所有 Markdown 格式的博客文章。
themes/：存放博客主题。
public/：Hexo 生成的网页文件，后续上传到 Cloudflare Page 的就是该文件夹内的文件。
第六步：修改 _config.yml 文件
为博客起个名字，相关修改都是简单的英文内容，若不明白可直接搜索教程。注意“:”后面要有空格，否则生成博客网页文件时容易报错。

第七步：创建第一篇博客文章
按 Win + R 组合键，输入 cmd 打开命令提示符窗口，使用 cd 命令进入博客安装文件夹路径，输入 hexo n "我的第一篇博客" 并回车，此时可在 source/_posts/ 文件夹里看到名为“我的第一篇博客”的 .md 格式文件，使用 Markdown 编辑器即可编辑，Windows 11 的记事本也支持 Markdown 编辑。

第八步：生成文件并上传到 Cloudflare Page
在命令提示符窗口输入 hexo g 生成文件。
将 public/ 文件夹下的所有文件打包成 zip 格式。
把 zip 压缩包拖到 Cloudflare 后台上传并应用，一般几秒钟后，博客就能流畅打开。
提示：很多新手在第二次新建文章时会报错，一定要先使用 cd 命令进入博客安装文件夹路径后再新建文章。

💬 最后说一句：
Hexo 是工具，Cloudflare Pages 是舞台，而你写的每一个字，才是博客真正的灵魂。
不必追求炫技，从一篇真实的文字开始——你已经是个博主了。

🚀 Happy Blogging on Cloudflare Pages!
